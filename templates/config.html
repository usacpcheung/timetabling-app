Weights and Toggles control solver priorities and feasibility rules. Weights are relative (larger means stronger preference). Values are internally scaled for the solver; keep using the same units.

  • Min/Max lessons per student: Global defaults. Individual student overrides (Advanced) take precedence.
  • Teacher min/max lessons: Global defaults used when a teacher’s own Min/Max is left blank.
  • Allow repeated lessons: If off, each (student, teacher, subject) can occur at most once.
  • Max repeats: Upper limit when repeats are allowed (applies per student–teacher–subject).
  • Allow consecutive repeats: If off, repeated lessons for the same triple cannot be in adjacent slots.
  • Prefer consecutive repeats: Adds a bonus to schedule back‑to‑back repeats (only takes effect when consecutive is allowed).
  • Allow different teachers per subject: If off, a student can study a subject with at most one teacher; repeats with that same teacher are still allowed.
  • Require all subjects: Ensures each active student receives at least one lesson for every listed subject.

Objective Weights influence what the solver prefers when multiple schedules are feasible:

  • consecutive_weight: Reward for consecutive repeats (typical 0–10). 0 disables the preference.
  • balance_weight: Penalty on the gap between most/least busy teachers (typical 0–10). 0 disables load balancing.
  • attendance_weight: Extra weight for subjects where a student (or group median) is below the subject’s Min% target (typical 0–20).
  • well_attend_weight: Weight to use for subjects already meeting the Min% target (typical 0.5–2.0).
  • group_weight: Multiplier for group lessons to bias scheduling toward groups (typical 1.0–3.0). Set to 0 to remove the bias.

Attendance Priority (optional):

  • Enable Attendance Priority to favor subjects where a student’s attendance % is below the Subjects → Min % threshold.
  • Weights used are attendance_weight (below target) and well_attend_weight (at/above target).
  • For groups, the median attendance among members is used to decide the weight.
  • Note: Combining Require All Subjects and Attendance Priority can slow solving.

Locations (optional):

  • If no locations exist, lessons are scheduled without location constraints.
  • If locations exist, each lesson is assigned a location; at most one lesson per location per slot.
  • Use Student/Group → Locations to restrict which locations can be used for that entity.

Unavailability & Blocking:

  • Teacher Unavailability marks slots a teacher cannot teach.
  • Student Advanced → Blocked Slots prevents any lesson for that student in the selected slots.
  • Student → Forbidden Teachers blocks specific teachers for that student. Group pseudo‑students inherit the union of member blocks.

Fixed Assignments:

  • Force specific teacher/student(or group)/subject/slot combinations.
  • When both a student and a group are provided on creation, the entry is saved for the group only if group_weight > 0 (or no student is chosen).

Per‑Student Overrides (Advanced dialog):

  • Min/Max lessons, Allow Repeats, Max Repeats, Allow/Prefer Consecutive, Allow Different Teachers per Subject, Blocked Slots.
  • These settings override the global defaults for that student only.
